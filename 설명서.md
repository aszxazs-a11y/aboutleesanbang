## 1. 프로젝트 구조 이해하기

### 폴더 구조

```
4일차/
├── portfolio_project/     ← Django 프로젝트 설정 폴더
│   ├── __init__.py       ← 패키지 인식 파일
│   ├── settings.py       ← 프로젝트 전체 설정
│   ├── urls.py           ← URL 경로 설정
│   ├── wsgi.py           ← 배포용 서버 연결
│   └── asgi.py           ← 비동기 서버 연결
│
├── portfolio/            ← 실제 앱 코드 폴더
│   ├── __init__.py       ← 패키지 인식 파일
│   ├── admin.py          ← 관리자 페이지 설정
│   ├── apps.py           ← 앱 설정
│   ├── models.py         ← 데이터베이스 구조 정의
│   ├── views.py          ← 페이지 로직 (어떤 데이터를 보여줄지)
│   └── urls.py           ← 앱의 URL 경로 설정
│
├── templates/            ← HTML 템플릿 폴더
│   ├── base.html         ← 모든 페이지의 기본 틀
│   ├── home.html         ← 홈페이지
│   ├── about.html        ← 소개 페이지
│   └── works/
│       ├── list.html     ← 작업 목록 페이지
│       └── detail.html   ← 작업 상세 페이지
│
├── static/               ← 정적 파일 폴더
│   ├── css/
│   │   └── style.css     ← 디자인 스타일
│   └── js/
│       └── main.js       ← 인터랙티브 기능
│
├── media/                ← 업로드된 파일 저장 폴더
├── manage.py             ← Django 관리 명령어 실행 파일
├── requirements.txt      ← 필요한 패키지 목록
├── build.sh              ← Render 배포 스크립트
└── 설명서.md             ← 이 문서
```

### 각 파일의 역할 (쉬운 비유)

| 파일/폴더 | 비유 | 설명 |
|-----------|------|------|
| settings.py | 건물 설계도 | 프로젝트의 모든 설정을 담고 있습니다 |
| models.py | 서류 양식 | 데이터를 어떤 형태로 저장할지 정의합니다 |
| views.py | 직원 | 요청을 받고 적절한 데이터를 가져와 응답합니다 |
| urls.py | 안내 데스크 | 주소를 보고 어떤 직원(view)에게 보낼지 결정합니다 |
| templates/ | 편지지 | 데이터를 담아 보여줄 HTML 템플릿입니다 |
| static/ | 인테리어 | CSS(디자인), JS(동작) 파일입니다 |
| admin.py | 관리자 사무실 | 관리자 페이지 설정입니다 |

## 2. 로컬에서 실행하기

### 2-1. 가상환경 만들기 (권장)

가상환경은 프로젝트별로 패키지를 분리하여 관리하는 방법입니다.
(선택사항이지만 강력히 권장합니다)

**Windows (명령 프롬프트 또는 PowerShell):**

```bash
# 1. 프로젝트 폴더로 이동
cd "C:\Users\이한길\OneDrive\Desktop\4일차"

# 2. 가상환경 생성 (venv라는 폴더가 만들어집니다)
python -m venv venv

# 3. 가상환경 활성화
# 명령 프롬프트:
venv\Scripts\activate

# PowerShell:
venv\Scripts\Activate.ps1

# 활성화되면 프롬프트 앞에 (venv)가 표시됩니다
```

**PowerShell에서 오류가 나는 경우:**
```powershell
# 실행 정책 변경 (관리자 권한으로 실행)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 2-2. 필요한 패키지 설치

```bash
# requirements.txt에 있는 모든 패키지 설치
pip install -r requirements.txt
```

설치되는 패키지:
- **Django**: 웹 프레임워크 (웹사이트를 만드는 도구)
- **gunicorn**: 웹 서버 (배포할 때 사용)
- **whitenoise**: 정적 파일 서빙 (CSS, JS 파일 제공)
- **Pillow**: 이미지 처리 (프로필 사진, 작업 이미지 등)

### 2-3. 데이터베이스 생성

Django는 데이터베이스 구조를 코드로 관리합니다.
아래 명령어로 실제 데이터베이스를 생성합니다.

```bash
# 1. 마이그레이션 파일 생성 (데이터베이스 설계도)
python manage.py makemigrations

# 2. 마이그레이션 적용 (실제 테이블 생성)
python manage.py migrate
```

성공하면 프로젝트 폴더에 `db.sqlite3` 파일이 생성됩니다.
이 파일이 데이터베이스입니다.

### 2-4. 관리자 계정 생성

관리자 페이지에 로그인할 계정을 만듭니다.

```bash
python manage.py createsuperuser
```

입력 항목:
- **사용자 이름**: admin (원하는 이름)
- **이메일 주소**: (선택사항, Enter로 건너뛰기 가능)
- **비밀번호**: 8자 이상 (입력해도 화면에 표시되지 않음)
- **비밀번호 확인**: 같은 비밀번호 다시 입력

### 2-5. 개발 서버 실행

```bash
python manage.py runserver
```

성공하면 다음과 같이 표시됩니다:
```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

### 2-6. 웹브라우저에서 확인

- **홈페이지**: http://127.0.0.1:8000/
- **작업 목록**: http://127.0.0.1:8000/works/
- **소개 페이지**: http://127.0.0.1:8000/about/
- **관리자 페이지**: http://127.0.0.1:8000/admin/

서버를 종료하려면 터미널에서 `Ctrl + C`를 누르세요.

## 3. 관리자 페이지 사용하기

### 3-1. 관리자 페이지 접속

1. http://127.0.0.1:8000/admin/ 접속
2. 생성한 관리자 계정으로 로그인

### 3-2. 데이터 추가하기

#### 사이트 설정 (Site settings)

홈페이지의 히어로 섹션 내용을 설정합니다.

1. "사이트 설정" 클릭
2. "사이트 설정 추가" 클릭
3. 내용 입력:
   - **히어로 제목**: 홈페이지에 크게 표시될 제목
   - **히어로 설명**: 제목 아래 표시될 설명
   - **간략 소개**: 홈페이지 하단의 소개글
4. "저장" 클릭

#### 프로필 (Profile)

소개 페이지에 표시될 정보입니다.

1. "프로필" 클릭
2. "프로필 추가" 클릭
3. 내용 입력:
   - **이름**: 본인 이름
   - **이메일**: 연락받을 이메일
   - **자기소개**: 상세 소개글
   - **프로필 사진**: 사진 업로드
4. "저장" 클릭

#### SNS 링크 (Sns 링크)

소개 페이지에 표시될 SNS 링크입니다.

1. "Sns 링크" 클릭
2. "Sns 링크 추가" 클릭
3. 내용 입력:
   - **플랫폼**: Instagram, Twitter, GitHub 등
   - **URL**: SNS 프로필 주소
   - **순서**: 표시 순서 (0이 가장 먼저)
4. "저장" 클릭

#### 카테고리 (Category)

작업물을 분류할 카테고리입니다.

1. "카테고리" 클릭
2. "카테고리 추가" 클릭
3. 내용 입력:
   - **카테고리명**: 웹 디자인, 일러스트 등
   - **순서**: 표시 순서
4. "저장" 클릭

#### 작업 (Work)

포트폴리오의 핵심인 작업물입니다.

1. "작업" 클릭
2. "작업 추가" 클릭
3. 내용 입력:
   - **제목**: 작업물 제목
   - **카테고리**: 위에서 만든 카테고리 선택
   - **썸네일**: 목록에 표시될 대표 이미지
   - **설명**: 작업물 상세 설명
   - **외부 링크**: 관련 사이트 주소 (선택사항)
4. 하단의 "작업 이미지"에서 추가 이미지 업로드 가능
5. "저장" 클릭

## 4. Render에 배포하기

Render는 웹사이트를 인터넷에 공개할 수 있는 무료 호스팅 서비스입니다.

### 4-1. 사전 준비

#### GitHub 계정 및 저장소 만들기

1. https://github.com 에서 계정 생성
2. 새 저장소(Repository) 생성:
   - 저장소 이름: portfolio (원하는 이름)
   - Public 선택
   - "Create repository" 클릭

#### Git 설치 (Windows)

1. https://git-scm.com/download/win 에서 다운로드
2. 설치 (기본 옵션으로 진행)

#### 프로젝트를 GitHub에 업로드

```bash
# 프로젝트 폴더에서 실행
cd "C:\Users\이한길\OneDrive\Desktop\4일차"

# Git 초기화
git init

# 모든 파일 추가
git add .

# 첫 커밋
git commit -m "Initial commit"

# GitHub 저장소 연결 (본인의 저장소 주소로 변경)
git remote add origin https://github.com/본인아이디/portfolio.git

# 업로드
git branch -M main
git push -u origin main
```

### 4-2. Render 설정

1. https://render.com 에서 계정 생성 (GitHub 계정으로 가입 권장)

2. 대시보드에서 "New +" → "Web Service" 클릭

3. GitHub 저장소 연결:
   - "Connect a repository" 클릭
   - GitHub 계정 연결
   - 포트폴리오 저장소 선택

4. 서비스 설정:
   - **Name**: portfolio (원하는 이름)
   - **Region**: Singapore (가장 가까운 지역)
   - **Branch**: main
   - **Runtime**: Python 3
   - **Build Command**: `./build.sh`
   - **Start Command**: `gunicorn portfolio_project.wsgi`

5. 환경 변수 설정 (Advanced 섹션):
   - "Add Environment Variable" 클릭
   - **SECRET_KEY**: 아무 긴 문자열 (예: my-super-secret-key-12345)
   - **DEBUG**: False
   - **PYTHON_VERSION**: 3.11.0

6. "Create Web Service" 클릭

### 4-3. 배포 완료 후

배포가 완료되면 `https://portfolio-xxxx.onrender.com` 형태의 URL이 생성됩니다.

**주의사항:**
- Render 무료 플랜은 15분 동안 접속이 없으면 서버가 대기 상태로 전환됩니다
- 대기 상태에서 첫 접속 시 30초~1분 정도 로딩이 걸릴 수 있습니다
- SQLite 데이터베이스는 배포할 때마다 초기화됩니다 (영구 데이터 저장을 원하면 PostgreSQL 사용 필요)

## 5. 자주 묻는 질문

### Q: 서버 실행 시 "포트가 이미 사용 중" 오류가 나요

다른 포트를 사용해서 실행하세요:
```bash
python manage.py runserver 8080
```
그 다음 http://127.0.0.1:8080/ 으로 접속

### Q: 이미지가 표시되지 않아요

1. 이미지 파일이 `media/` 폴더에 있는지 확인
2. 개발 서버가 실행 중인지 확인
3. settings.py의 MEDIA_URL, MEDIA_ROOT 설정 확인

### Q: CSS가 적용되지 않아요

1. 정적 파일 수집:
   ```bash
   python manage.py collectstatic
   ```
2. 브라우저 캐시 삭제 (Ctrl + Shift + R)

### Q: 관리자 페이지에 접속이 안 돼요

1. 서버가 실행 중인지 확인
2. 관리자 계정을 생성했는지 확인:
   ```bash
   python manage.py createsuperuser
   ```

### Q: 마이그레이션 오류가 나요

마이그레이션 파일을 삭제하고 다시 시도:
```bash
# portfolio/migrations/ 폴더에서 __init__.py 빼고 모두 삭제
# db.sqlite3 파일 삭제
python manage.py makemigrations
python manage.py migrate
```

### Q: 가상환경을 비활성화하려면?

```bash
deactivate
```
