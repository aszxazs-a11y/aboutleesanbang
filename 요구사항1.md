나는 포트폴리오 사이트 만들려고 해.

[주요 기능]

- 홈: 인터랙티브 히어로 섹션, 간략 소개 텍스트
- 작업 목록: 썸네일 그리드 레이아웃, 카테고리 태그 표시, 호버 시 프로젝트 정보 표시
- 작업 상세: 이미지 갤러리(여러 장), 프로젝트 설명, 외부 링크, 좋아요 기능, 댓글 기능(닉네임+비밀번호+내용), 이전/다음 네비게이션
- 소개: 프로필(이름, 사진, 바이오), 이메일(복사 기능), SNS 링크
- 글로벌: 커스텀 커서, 다크/라이트 모드 토글, 페이지 전환 애니메이션, 로딩 인터랙션

[URL 설계]
/ 홈 → 히어로(인터랙티브), 간략 소개
/works/ 작업 목록 → 썸네일 그리드, 카테고리 태그, 호버 인터랙션
/works/int:id/ 작업 상세 → 이미지 갤러리, 설명, 외부 링크, 좋아요, 댓글, 이전/다음
/about/ 소개 → 프로필, 이메일(복사), SNS 링크

[DB 설계]

- SiteSettings: hero_title(문자열), hero_description(텍스트), short_intro(텍스트)
- Profile: name(문자열), bio(텍스트), email(문자열), profile_image(이미지)
- SocialLink: platform(문자열), url(문자열), order(정수)
- Category: name(문자열), order(정수)
- Work: title(문자열), description(텍스트), thumbnail(이미지), external_link(문자열), category_id(FK→Category), created_at(날짜시간)
- WorkImage: work_id(FK→Work), image(이미지), order(정수)
- Comment: work_id(FK→Work), author_name(문자열), password(문자열), content(텍스트), created_at(날짜시간)
- Like: work_id(FK→Work), created_at(날짜시간)

[모델 관계]

- Category → Work (1:N)
- Work → WorkImage (1:N)
- Work → Comment (1:N)
- Work → Like (1:N)

Django로 구현해줘. 배포는 render에서 진행할 예정인데 requirements.txt 파일이 필요하고 render 서비스에서 배포할 때 처음에 'gunicorn 프로젝트명.wsgi'으로 시작하니 함께 설정해줘.

[사용자 현재 상황]
- 나는 코드를 처음 작성하는 사람이야. 비개발자 및 코딩지식 없는 사람이 이해할 수 있는 수준의 상세한 설명서가 매우 상세하게 있어야 해.

- 컴퓨터에는 Pyton이 설치 되어 있어.

- 각 파일별로 어떤 역할을 하는지 주석으로 설명을 달아줘.
